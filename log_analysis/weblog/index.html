    Web日志字段解析

    本文以以主流的nginx为例

    nginx日志字段

    配置access log

    http_log_module文档

    upstream_module文档

    默认日志字段

    字段 含义 示例
    remote_addr 客户端地址 10.10.10.1
    remote_user 客户端用户名
    time_local 时间戳 16/Jun/2019:23:29:50 -0400
    request 请求的URI和协议 GET /favicon.ico HTTP/1.1
    status 返回状态码 404
    body_bytes_sent 响应body字节数 3650
    http_referer referer信息 http://10.10.10.14/
    http_user_agent UA信息 Mozilla/5.0 (Macintosh; Intel Mac OS X 10_14_5) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/75.0.3770.90 Safari/537.36
    http_x_forwarded_for 客户端IP 192.168.1.1

    其它可使用字段

    字段 含义 示例
    - 占位符 -
    scheme 请求协议 http
    request_method http请求方法 GET
    request_time 请求处理时间 0.000
    request_length 请求的长度 571
    bytes_sent 响应总字节数 175
    server_name 虚拟主机名称
    http_host IP或域名(包括端口) 10.10.10.14:81
    host IP或域名(不包括端口) 10.10.10.14
    request_body 请求的body
    upstream_status upstream状态 200
    upstream_addr 回源IP
    upstream_connect_time 与服务器连接所花费的时间
    upstream_response_time 后端处理时间
    ssl_protocol SSL协议版本 TLSv1
    ssl_cipher 交换数据中的算法 RC4-SHA

    nginx透传客户端真实IP

    当经过N个nginx代理时，Server端如何拿到Client端端真实IP？

    # 环境

client_ip：10.10.10.1

proxy_server_1：10.10.10.16
proxy_server_2：10.10.10.17

web_server：10.10.10.18

    使用X-Forwarded-For + realip模块

    使用realip模块可以解决该问题。

    查看nginx的编译参数：nginx -V（默认是安装了realip模块的）

    # set_real_ip_from：表示从何处获取真实IP，只认可自己信赖的IP，可以是网段，也可以设置多个。
# real_ip_header：表示从哪个header属性中获取真实IP。
# real_ip_recursive：递归检索真实IP，如果从X-Forwarded-For中获取，则需要递归检索；如果中X-Real-IP中获取，无需递归。

    配置nginx

    # proxy_server_1的nginx配置
location / {
    proxy_pass http://10.10.10.17;
    proxy_set_header Host $http_host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}

# proxy_server_2的nginx配置
location / {
    proxy_pass http://10.10.10.18;
    proxy_set_header Host $http_host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}

# web_server的nginx配置，在server中新增
set_real_ip_from 10.10.10.16;
set_real_ip_from 10.10.10.17;
real_ip_header X-Forwarded-For;
real_ip_recursive on;

    使用X-Forwarded-For + 安全设置

    在第一个代理处处理用户传递的XFF，达到忽略用户传递的XFF值的效果。

location / {
    proxy_pass http://10.10.10.17;
    proxy_set_header X-Forwarded-For $remote_addr;
}

# proxy_server_2的nginx配置
location / {
    proxy_pass http://10.10.10.18;

    nginx打印request_body 与 response_body

    打印request_body

    # web_server配置
# The variable’s value is made available in locations processed by the proxy_pass, fastcgi_pass, uwsgi_pass, and scgi_pass directives when the request body was read to a memory buffer.
# 意味着：只有location中用到proxy_pass、fastcgi_pass、scgi_pass命令时，request_body才有值。

log_format main $request_body # 我这里是简写

location ~ \.php$ {
    root           /usr/share/nginx/html;
    fastcgi_pass   127.0.0.1:9000;
    fastcgi_index  index.php;
    fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
    include        fastcgi_params;
}

    打印response_body

    需要配置lua模块，参考：nginx标签下的文章

    # web_server配置
    log_format main $response_body # 我这里是简写
    server {
        listen       80;
        server_name  localhost;
        lua_need_request_body on;
        set $response_body "";
        body_filter_by_lua '
            local response_body = string.sub(ngx.arg[1],1,1000)
            ngx.ctx.buffered = (ngx.ctx.buffered or "") ..  response_body
            if ngx.arg[2] then
                ngx.var.response_body = ngx.ctx.buffered
            end
        ';
        location ~ \.php$ {
            root           /usr/local/nginx/html;
            fastcgi_pass   127.0.0.1:9000;
            fastcgi_index  index.php;
            fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
            include        fastcgi_params;
        }
    }

    注意事项

    • request_body 与 response_body 如果在上传文件或者下载文件时，body内容会很大，需要考虑对性能和存储的影响。
    • 对于timestamp参数，可以不实用time_local而使用time_iso8601。
    • 如果运维对于网络的性能有要求，可以考虑使用$tcpinfo_rtt, $tcpinfo_rttvar, $tcpinfo_snd_cwnd, $tcpinfo_rcv_space。
